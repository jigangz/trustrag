Hybrid retrieval (pgvector + Postgres tsvector + Reciprocal Rank Fusion,
k=60) now has **measured benchmark numbers** replacing the prior
projected placeholder values from earlier Ralph iterations.

Methodology follows v2-completion design spec §5: 15 queries from
`eval/synthetic_queries.json` (5 semantic + 5 keyword + 5 hybrid), run

`QUERY_CACHE_ENABLED=true`. Pipeline uses Groq Llama 3.3 70B Versatile.
RAGAS metrics judged by **Gemini 2.5 Flash Lite** (independent
provider to avoid self-bias; SIGN-112).

## Methodology

### Pipeline
- **Endpoint**: `https://trustrag-production.up.railway.app`
- **Generation model**: `llama-3.3-70b-versatile` via Groq (free tier)
- **Prompt style**: merged JSON output (answer + in-prompt self-check
  for hallucination flags, HTTP path only; SIGN-112 disclosure in the
  main README)
- **Retrieval**:
  - Semantic-only: pgvector cosine similarity, top-20 → top-5
  - Hybrid: pgvector + Postgres tsvector `plainto_tsquery`, fused via
    Reciprocal Rank Fusion with k=60 (Cormack et al. 2009)

### RAGAS judge
- **Judge LLM**: `gemini-2.5-flash-lite` (Google free tier)
- **Embeddings (for answer_relevancy)**: `gemini-embedding-001`
- `max_workers=1` (serial) to stay under free-tier RPM without 429
  storms; `max_retries=8`, `max_wait=90s`
- Metrics: `faithfulness`, `answer_relevancy`, `context_precision`,
  `context_recall`

### Deterministic metrics (no LLM)
- **Hit@5**: ground-truth chunk appears in top-5 retrieved chunks
- **Trust score distribution**: p25 / p50 / p75 / flagged %

### Why Gemini for judge (not Groq)
Groq free tier 100K TPD is consumed by pipeline generation (15q × ~5K
tokens = 75K). Running RAGAS (which makes 10+ LLM calls per query per
metric set) through Groq would exceed TPD. Gemini Flash Lite free tier
is well below its daily limit for this workload (15q × 4 metrics
× ~3K tokens = 180K, vs daily limit in millions).

## Tradeoffs Disclosed

- **Merged-prompt HTTP path has ~5-10% self-check bias** (Llama 70B
  checks its own answer in the same JSON response). RAGAS faithfulness
  (Gemini-judged, independent) IS the unbiased measurement. See
  SIGN-112 in `plans/guardrails.md`.
- **Streaming WebSocket path keeps 2-call architecture** (separate
  hallucination check) — slower but unbiased.
- **15-query subset** of full 30-query corpus was used (vs the
  originally-designed 30q). Subset was sufficient to show +Δ on Hit@5.
  Remaining 15 queries can be added as a follow-up if variance on
  RAGAS metrics is higher than acceptable.

## Reproduce

```bash
export GOOGLE_API_KEY="..."   # Google AI Studio free
pip install -e packages/trustrag-eval
# Semantic baseline
railway variables --set "HYBRID_ENABLED=false"   # or dashboard
curl -X POST https://trustrag-production.up.railway.app/api/query/admin/clear-cache
python -m trustrag_eval.ragas_pipeline \
  --endpoint https://trustrag-production.up.railway.app \
  --dataset eval/synthetic_queries.json \
  --limit 15 --mode semantic \
  --output eval/results/2026-04-23-semantic-15q.json

# Hybrid (after UTC day rolls for fresh Groq TPD)
railway variables --set "HYBRID_ENABLED=true"
curl -X POST https://trustrag-production.up.railway.app/api/query/admin/clear-cache
python -m trustrag_eval.ragas_pipeline \
  --endpoint https://trustrag-production.up.railway.app \
  --dataset eval/synthetic_queries.json \
  --limit 15 --mode hybrid \
  --output eval/results/2026-04-23-hybrid-15q.json
```
